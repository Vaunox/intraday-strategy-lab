## 0. Two INDEPENDENT studies (not a both-owed dichotomy)

A volatility filter has no standalone edge to measure — it only means something applied on top of a directional signal. So P3.8 tests the volatility-filter thesis as **two self-contained, blind, regime-conditional strategies**, defined whole (regime + signal together), run as **two INDEPENDENT standalone studies** — each its own verdict, each can pass or fail on its own:

- **C1 — Expansion → Breakout** (`vol_expansion_breakout`).
- **C2 — Contraction → Mean-Reversion** (`vol_contraction_reversion`).

**They are NOT a both-owed dichotomy.** C1 and C2 are *compatible, independent* strategies operating in *disjoint* regimes; both passing would be a coherent "breakouts work in expansion AND reversion works in contraction" (the filter thesis confirmed), NOT a paradox. Forcing dichotomy / "if both pass, contradiction" logic on them would be false framing. Running **both** serves one purpose only: **it removes all selection ambiguity** — neither regime-direction is cherry-picked from prior results — and answers the symmetric question (does regime-conditioning help either family?). Either passing → STOP + flag + verdict-grade path.

## 1. Hypotheses (falsifiable) — per study

On the frozen large-cap **panel** at 5-minute bars, each study produces a **positive net-of-cost edge** clearing the **two-part panel verdict** (aggregate seven-point gate on the equal-weight portfolio AND breadth: median per-symbol CPCV path-Sharpe > 1.0 and ≥ 60% of symbols individually positive), net of the full Indian round-trip cost, realized-frequency annualized. **Falsified if** the panel verdict is KILL/INSUFFICIENT, or the cost-viability pre-check finds the median gross per-trade move below the modeled round-trip cost.

## 2. Economic rationale (mechanism, not pattern) — PURELY a-priori

Both reasons are true **before any result** — no reference to prior outcomes.

- **C1 (Expansion → Breakout):** a breakout **mechanically requires volatility to be expanding** — a trading range resolves into a directional move only when dispersion widens; a "break" in *contracting* volatility has no move to break into and reverts. So gate breakout entries to an expanding-vol regime.
- **C2 (Contraction → Mean-Reversion):** mean-reversion **requires a quiet range to oscillate within** — in *contracting* volatility there is no directional energy, so price reverts to the mean; in expanding volatility a fade fights a real move. So gate reversion entries to a contracting-vol regime.

## 3. Specs (event → entry → exit/holding → position)

Both in [`src/lab/research/strategies/volatility_filters.py`](../../src/lab/research/strategies/volatility_filters.py); both gate a directional rule by a **shared ATR-ratio regime** (`indicators.atr_ratio` = `atr(atr_short) / atr(atr_long)`, dimensionless, self-normalizing per symbol — directly comparable across the panel without a rank; causal — both ATRs from completed prior bars).

- **C1 — `VolExpansionBreakoutSpec`** (`vol_expansion_breakout`): when `atr_ratio > 1.0` (recent ATR above the longer baseline → expanding), trade the prior-N-bar intraday breakout — `close > intraday_donchian_high` ⇒ LONG, `close < low` ⇒ SHORT; ride to the opposite in-regime break or 15:20 square-off. Out of regime (`≤ 1.0`) → no new entry.
- **C2 — `VolContractionReversionSpec`** (`vol_contraction_reversion`): when `atr_ratio < 1.0` (contracting), fade the intraday z-score — `intraday_zscore > +entry_z` ⇒ SHORT, `< −entry_z` ⇒ LONG, exit when `|z| ≤ exit_z`; wide-entry/narrow-exit hysteresis. Out of regime (`≥ 1.0`) → no new entry.
- Each day opens flat; 15:20 square-off; fixed unit size.

**Causal / no-lookahead (hard precondition, both studies).** The regime, signal, and position at bar `i` use only bars `≤ i` (ATR and `intraday_zscore` are trailing/prior-only; `intraday_donchian` excludes the current bar). The shared **`atr_ratio` primitive has its own prefix-invariance test** (`talib.ATR` confirmed prefix-invariant), and **each spec has a prefix-invariance no-lookahead test** — all **hard preconditions**: if any fails, that study does not run.

## 4. Pre-committed parameters (FROZEN before the run, set BLIND)

| Study | Regime (shared) | Directional | Swept params | Fixed blind |
|---|---|---|---|---|
| C1 | `atr_ratio(20,100) > 1.0` | intraday breakout | `breakout_lookback` **20** ±5, `atr_long` **100** ±20 | `atr_short` **20** |
| C2 | `atr_ratio(20,100) < 1.0` | z-score fade | `entry_z` **2.0** ±0.5, `atr_long` **100** ±20 | `atr_short` **20**, `exit_z` **0.5**, `lookback` **20** |

**5 configs each** (base + two ±one-step params). **Chosen BLIND, a-priori:** `atr_short` 20 (~intraday, ~100 min) vs `atr_long` 100 (~1.3 sessions baseline) — recent dispersion vs a longer baseline; the ratio boundary at **1.0** ("above/below this stock's own baseline") is the natural, threshold-free split (no arbitrary cutoff). **Not** derived from any regime-performance breakdown (none seen; criterion-7 was "positive in no bucket"). The `atr_long`± sweep tests that the result is not cliff-sensitive to the boundary. **One regime condition each; no stacking.** Registry entries pinned by `tests/unit/test_registry.py`.
